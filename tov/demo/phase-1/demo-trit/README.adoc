[[three-layers]]
== Three-Layer Architecture

The libtrit library organizes ternary computing into three cognitive layers:

[[layer-matter]]
=== Layer 1: MATTER (trit.h)

*What exists* â€” individual trit operations.

[cols="2,4",options="header"]
|===
| Function | Purpose

| `trit_create(int)` | Create trit from integer (-1, 0, +1)
| `trit_valid(trit_t)` | Check if trit is valid
| `trit_value(trit_t)` | Extract integer value
| `trit_negate(trit_t)` | Flip sign (-1â†”+1, 0 unchanged)
| `trit_add(trit_t, trit_t)` | Add two trits (no carry)
| `trit_multiply(trit_t, trit_t)` | Multiply two trits
|===

[[layer-space]]
=== Layer 2: SPACE (dimension.h + pack.c)

*Where it exists* â€” pack, navigate, map.

[cols="2,4",options="header"]
|===
| Function | Purpose

| `trit5_pack(trit_t[5])` | Pack 5 trits into trit5_t (0-242)
| `trit5_unpack(trit5_t, trit_t[5])` | Unpack trit5_t to 5 trits
| `trit5_is_spare(trit5_t)` | Check if spare state (243-255)
| `trit_to_dimension(trit_t)` | Map trit to dimension
| `dimension_path(dim, dim)` | Navigate between dimensions
|===

[[layer-time]]
=== Layer 3: TIME (temporal.h)

*When it exists* â€” temporal state cognition.

[cols="2,4",options="header"]
|===
| Function | Purpose

| `trit_to_temporal(trit_t)` | Map trit to temporal state
| `temporal_create_base(temporal_t)` | Create base temporal state
| `temporal_create_compound(temporal_t, temporal_t)` | Create compound state
| `temporal_is_compound(temporal_state_t)` | Check if compound
