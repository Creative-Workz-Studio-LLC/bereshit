[[the-mapping]]
== The Mapping

MillenniumOS thinks in ternary (-1, 0, +1) but runs on binary hardware (0, 1). The mapping layer bridges this gap.

=== Ternary Values

[cols="^1,2,3",options="header"]
|===
| Value | Name | Building Block Role

| *-1*
| Negative
| Break Down (toward foundation)

| *0*
| Neutral
| Foundation/Anchor (the immutable center)

| *+1*
| Positive
| Build Up (from foundation)
|===

The building block method IS the operating logic: `-1 + 0 + 1 = 0` â€” balanced system.

=== Trit-to-Byte Mapping

**Single byte (8 bits) holds 5 trits:**

[cols="2,2,3",options="header"]
|===
| System | Capacity | Calculation

| *Ternary (5 trits)*
| 243 states
| 3^5 = 243

| *Binary (8 bits)*
| 256 states
| 2^8 = 256

| *Gap*
| 13 states
| 256 - 243 = 13
|===

**The 13 WEB variants fill the exact gap.** Not coincidence â€” design.

=== Packed vs Unpacked

[cols="1,2,2,2",options="header"]
|===
| Mode | Representation | Use Case | Trade-off

| *Unpacked*
| 1 byte per trit
| Computation (fast access)
| Simple but wasteful

| *Packed*
| 5 trits per byte
| Storage/transmission
| Efficient but requires encode/decode
|===

The system breathes â€” compressing and expanding as data moves:

```
Storage (packed) â†’ decompress â†’ Memory (unpacked) â†’ compute â†’ compress â†’ Transmission (packed)
```
