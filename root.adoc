[[structure]]
== Structure & Flow

The ternary structure follows the Genesis creation pattern — God created by separation (בָּדַל _badal_), establishing boundaries before content.

[[flow]]
=== The Ternary Flow

----
         bereshit (origin ∞)
              |
    +---------+---------+
    |         |         |
  void      word       tov
 (entry)    (def)    (exit)
   -1         0        +1
----

. *void/* — Entry gate (null) — where things enter awaiting classification
. *word/* — Definitions (1) — source of truth, contains scripture/
. *tov/* — Exit gate (complete) — where finished work exits to the world

[TIP]
====
*Tight, not guided.* The ternary structure is foundational — three children exactly. Content within children can be adapted, but the root structure is fixed.
====

[%collapsible]
.Deep Dive: Why Ternary?
====
The ternary (-1/0/+1) model emerges from Genesis 1:

[cols="^1,2,3,3",options="header"]
|===
| Value | Name | Genesis Reference | Meaning

| -1
| *void*
| "without form, and void" (1:2)
| Entry state — formless, awaiting classification

| 0
| *word*
| "God said" (1:3, 6, 9...)
| Definition state — the spoken Word brings form

| +1
| *tov*
| "it was good" (1:4, 10, 12...)
| Completion state — evaluated and approved
|===

**The math:** `-1 + 0 + 1 = 0` — a balanced system. The ternary is complete; nothing can be added or removed without breaking the pattern.

**The flow:** External → void (entry) → word (definition) → tov (exit) → External
====

[[health-scoring]]
=== Health Scoring

Bereshit uses the *Base100*footnote:[Base100 = health scoring system with range -100 to +100, where ternary thresholds (-50/+50) determine state] health scoring system.

[cols="^1,2,3",options="header"]
|===
| State | Threshold | Meaning

| *Granted (+1)*
| Above +50
| Filesystem structure healthy

| *Deferred (0)*
| Within ±50
| Filesystem needs attention

| *Denied (-1)*
| Below -50
| Filesystem structure broken
|===

[%collapsible]
.Deep Dive: How Health Scoring Works
====
Health scoring measures structural integrity, not content quality.

**What gets scored:**

[cols="^1,3,3",options="header"]
|===
| Weight | Check | Impact

| *Critical*
| Required files present (root.omni, .health)
| Missing = immediate Denied

| *Structural*
| Block boundaries intact
| Broken = major penalty

| *Consistency*
| Metadata matches reality
| Drift = minor penalty
|===

**The `.health` file:**

* Binary format (8 bytes) for machine efficiency
* Stores: health score, timestamp, version
* Created/updated by `bhealth` toolfootnote:[bhealth = Bereshit health tool - validates and initializes filesystem health]

**Why Base100?**

The range (-100 to +100) allows granular scoring while maintaining ternary alignment:

* Positive scores (+1 to +100) = healthy, varying degrees of excellence
* Zero = neutral, needs attention but not broken
* Negative scores (-1 to -100) = unhealthy, varying degrees of damage

The thresholds (+50/-50) create clear decision boundaries while the full range enables nuanced measurement.
====

[[biblical-foundation]]
== Biblical Foundation

[NOTE]
====
*Bereshit (בְּרֵאשִׁית)* means "In the beginning" — the foundation from which all creation flows.
====

_"In the beginning God created the heaven and the earth."_ — *Genesis 1:1*

*Applied:* Bereshit is the foundation from which all creation flows. The ternary structure (void/word/tov) reflects the Genesis pattern — from formless (void) to defined (word) to complete (tov). The root contains only structure, never content directly.

[%collapsible]
.Deep Dive: The Genesis Pattern in Detail
====
**Genesis 1 as Architectural Blueprint:**

The first chapter of Genesis reveals God's pattern for creation — not arbitrary, but purposeful structure emerging through separation (בָּדַל _badal_footnote:[badal (בָּדַל) = Hebrew "to separate, divide, distinguish" - God's primary creative action in Genesis 1]).

[cols="^1,2,3,3",options="header"]
|===
| Day | Action | Type | Bereshit Parallel

| 1
| Light from darkness
| Separation
| void/tov boundary

| 2
| Waters above/below
| Separation
| Structure before content

| 3
| Land/seas, vegetation
| Separation + filling
| word contains definitions

| 4-6
| Filling domains
| Content
| Children contain actual work

| 7
| Rest (שַׁבָּת _shabbat_)
| Completion
| tov = complete, ready for release
|===

**The Hebrew Insight:**

* *בְּרֵאשִׁית* (bereshit) = "In beginning" — no definite article, emphasizing origin itself
* *בָּרָא* (bara) = "created" — used only of God's creative work, not human making
* *אֱלֹהִים* (Elohim) = God — plural form with singular verbs, hinting at unified complexity

**Why This Matters for Filesystem:**

God created by *establishing boundaries before filling content*. Light existed before sun. Land was named before planted. The structure preceded the substance.

Bereshit follows this pattern:

1. **void/** exists as boundary before imports arrive
2. **word/** exists as container before definitions fill it
3. **tov/** exists as gate before completed work passes through

Content cannot exist without structure to hold it. This is not arbitrary organization — it's reflection of how creation itself works.

**The Anchor:**

Genesis 1:1 is the _anchor of anchors_. Every truth, when traced back far enough, halts here: God created. The filesystem's ternary structure isn't imposed philosophy — it's discovered pattern, revealed in the text that grounds all reality.
====
