[[overview]]
== Overview

Data Seeds contains templates for configuration and build files — Makefiles, YAML, JSON, TOML, and other structured data formats. These templates establish consistent patterns for project configuration.

This document follows the CWSfootnote:[CWS = CreativeWorkzStudio LLC] 5-block documentation structure.

[IMPORTANT]
====
*#Data templates prepare the field#* — Configuration files set up the environment before code runs. Well-structured data enables well-structured work.
====

[NOTE]
====
*Scope:* This document covers data and configuration templates. For code templates, see `code/`. For documentation templates, see `documentation/`.
====

'''

[[quick-start]]
=== Quick Start

[source,bash]
----
# 1️⃣ Enter data seeds
cd bereshit/word/seed/data/

[[structure]]
== Structure & Flow

Data Seeds is a child of word/seed/, containing format-specific subdirectories.

[[hierarchy]]
=== Folder Hierarchy

[mermaid]
----
flowchart TD
    S[seed/] --> D[data/]
    D --> BLD[build/]
    D --> CFG[config/]
    D --> SCH[schema/]
    D --> DOC[docker/]
----

Each format directory contains templates keyed appropriately:
- `DATA-MK-001-MK-project.mk`
- `DATA-YAML-001-YAML-config.yaml`
- `DATA-JSON-001-JSON-schema.json`
- `DATA-TOML-001-TOML-cargo.toml`

[[health-scoring]]
=== Health Scoring

Data Seeds uses the *Base100* health scoring system.

[cols="^1,2,3",options="header"]
|===
| State | Threshold | Meaning

| *Granted (1)*
| Above +50
| Templates current and validated

| *Deferred (0)*
| Within ±50
| Templates may need review

| *Denied (-1)*
| Below -50
| Templates invalid or incompatible
|===

==== Operations Scoring

[cols="3,^1,3",options="header"]
|===
| Condition | Score | Description

| *Templates valid*
| +20
| All templates parse correctly

| *Templates need updates*
| 0
| Format version changes may apply

| *Templates invalid*
| -20
| Parse or validation errors
|===

[[policy]]
== Policy

[[modification-rules]]
=== Modification Rules

[cols="^1,2,4",options="header"]
|===
| Level | Classification | Allowed Operations

| *1*
| *Safe*
| Copy templates to work/, Update template comments, Add new format templates

| *0*
| *Careful*
| Modify template structure, Change required fields

| *-1*
| *Never*
| Store project configs here (belongs in work/), Add secrets to templates
|===

[WARNING]
====
*Never put secrets in templates.* Data templates are patterns — actual configuration with secrets belongs in work/ and should be gitignored.
====
