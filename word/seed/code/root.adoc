[[overview]]
== Overview

Code Seeds contains templates for source code files — Go, Rust, C, shell scripts, and other programming languages. Each template follows the 4-block structure (METADATA → SETUP → BODY → CLOSING) adapted to the language's syntax.

This document follows the CWSfootnote:[CWS = CreativeWorkzStudio LLC] 5-block documentation structure.

[IMPORTANT]
====
*#Guided strictness means thoughtful adaptation#* — These templates are recommendations, not requirements. Adapt the structure to serve the code's purpose while maintaining the 4-block pattern where it aids clarity.
====

[NOTE]
====
*Scope:* This document covers code file templates. For data configuration templates, see `data/`. For documentation templates, see `documentation/`.
====

'''

[[structure]]
== Structure & Flow

Code Seeds is a child of word/seed/, containing language-specific subdirectories.

[[hierarchy]]
=== Folder Hierarchy

[mermaid]
----
flowchart TD
    S[seed/] --> C[code/]
    C --> GO[go/]
    C --> RS[rust/]
    C --> CL[c/]
    C --> SH[shell/]
    C --> ASM[asm/]
----

Each language directory contains templates keyed appropriately:
- `CODE-GO-001-GO-executable.go`
- `CODE-RS-001-RS-crate.rs`
- `CODE-C-001-C-source.c`
- `CODE-SH-001-SH-script.sh`

[[health-scoring]]
=== Health Scoring

Code Seeds uses the *Base100* health scoring system.

[cols="^1,2,3",options="header"]
|===
| State | Threshold | Meaning

| *Granted (1)*
| Above +50
| Templates current and tested

| *Deferred (0)*
| Within ±50
| Templates may need review

| *Denied (-1)*
| Below -50
| Templates broken or incompatible
|===

==== Operations Scoring

[cols="3,^1,3",options="header"]

| +20
| All templates build successfully

| *Templates need updates*

[cols="^1,2,4",options="header"]

| Level | Classification | Allowed Operations

| *1*
| *Safe*
