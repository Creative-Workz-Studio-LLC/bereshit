[[overview]]
== Overview

Documentation Seeds contains templates for written content — AsciiDoc, Markdown, reStructuredText, and other documentation formats. Each template follows the 5-block structure (METADATA → HEADER → CONTEXT → CONTENT → FOOTER) for comprehensive, layered documentation.

This document follows the CWSfootnote:[CWS = CreativeWorkzStudio LLC] 5-block documentation structure.

[IMPORTANT]
====
*#Documentation serves readers at every level#* — These templates enable layered reading: scan (5 sec), skim (1 min), read (full), study (deep), reference (quick lookup). Write for all readers.
====

[NOTE]
====
*Scope:* This document covers documentation templates. For code templates, see `code/`. For data configuration templates, see `data/`.
====

[[structure]]
== Structure & Flow

Documentation Seeds is a child of word/seed/, containing format-specific subdirectories.

[[hierarchy]]
=== Folder Hierarchy

[mermaid]
----
flowchart TD
    S[seed/] --> D[documentation/]
    D --> ADOC[B-word-seed-doc-asciidoc-base.adoc]
    D --> MD[B-word-seed-doc-markdown-*.md]
    D --> TYP[B-word-seed-doc-typst-base.typ]
----

Templates use B-prefix keying following bereshit conventions:

- `B-word-seed-doc-asciidoc-base.adoc` — Base AsciiDoc template
- `B-word-seed-doc-markdown-base.md` — Base Markdown template
- `B-word-seed-doc-markdown-index.md` — Index/navigation Markdown template
- `B-word-seed-doc-markdown-community.md` — Community-facing Markdown template
- `B-word-seed-doc-typst-base.typ` — Base Typst template

[[five-block]]
=== The 5-Block Pattern

Documentation templates follow the 5-block pattern:

[cols="^1,2,4",options="header"]
|===
| Block | Name | Purpose

| 1
| *METADATA*
| The catalog card — parseable identity for systems

| 2
| *HEADER*
| The cover — visible identity in 5 seconds

| 3
| *CONTEXT*
| The introduction — prepares understanding

| 4
| *CONTENT*
| The chapters — value organized for discovery

| 5
| *FOOTER*
| The appendix — resources and grounding
|===

[[health-scoring]]
=== Health Scoring

Documentation Seeds uses the *Base100* health scoring system.

[cols="^1,2,3",options="header"]
|===
| State | Threshold | Meaning

| *Granted (1)*
| Above +50
| Templates current and consistent

| *Deferred (0)*
| Within ±50
| Templates may need review

| *Denied (-1)*
| Below -50
| Templates incomplete or inconsistent
|===

[NOTE]
