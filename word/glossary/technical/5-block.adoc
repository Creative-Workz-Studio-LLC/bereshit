= 5-block

*Documentation Structure Pattern*

*Definition:* METADATA → HEADER → CONTEXT → CONTENT → FOOTER

[quote, Exodus 25:40 KJV]
____
And look that thou make them after their pattern, which was shewed thee in the mount.
____
link:../scripture/KJV/Exodus/Chapter_25/Verse_40.txt[Read verse ^]

== Use Case

All documentation - for human readers.

== The Blocks

[cols="1,2,2"]
|===
|Block |Purpose |Equivalent

|*METADATA*
|Machine-readable identity
|Same as code

|*HEADER*
|Human-readable identity (title, nav, status)
|—

|*CONTEXT*
|Prerequisites, definitions
|≈ SETUP

|*CONTENT*
|Main body, teachings
|≈ BODY

|*FOOTER*
|References, related docs
|≈ CLOSING
|===

== Why Different from 4-Block

Code is for compilers - they read METADATA directly.

Documentation is for humans - they need HEADER for quick visual orientation (title, navigation, status).
