= 4-block

*Code Structure Pattern*

*Definition:* METADATA → SETUP → BODY → CLOSING

[quote, Exodus 25:40 KJV]
____
And look that thou make them after their pattern, which was shewed thee in the mount.
____
link:../scripture/KJV/Exodus/Chapter_25/Verse_40.txt[Read verse ^]

== Use Case

ALL code follows this. Every file. Every time.

== The Blocks

[cols="1,2,2"]
|===
|Block |Purpose |Contains

|*METADATA*
|Identity, purpose, health scoring
|Core Identity (Biblical, CPI-SI, Authorship) → Interface → Operational

|*SETUP*
|Preparation for execution
|Imports → Constants → Variables → Types → Type Methods

|*BODY*
|Core implementation
|Org Chart → Helpers → Core Ops → Error Handling → Public APIs

|*CLOSING*
|Finalization and documentation
|Validation → Execution → Cleanup → Documentation
|===

== Core Principle

Forces intentional design.

If something doesn't fit, that's architectural feedback. Extract and orchestrate - don't create v2 files.
