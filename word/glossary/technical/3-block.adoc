= 3-block

*Data Structure Pattern*

*Definition:* METADATA → CONTENT → CLOSING

[quote, Exodus 25:40 KJV]
____
And look that thou make them after their pattern, which was shewed thee in the mount.
____
link:../scripture/KJV/Exodus/Chapter_25/Verse_40.txt[Read verse ^]

== Use Case

Simple data files - configs, settings, structured data.

== The Blocks

[cols="1,2"]
|===
|Block |Purpose

|*METADATA*
|Identity and schema information

|*CONTENT*
|The actual data

|*CLOSING*
|Validation and references
|===

== Why Simpler

Data files don't need:

* *SETUP* - no imports/variables to prepare
* *HEADER* - no human navigation needed for machine-readable data

== Template Location

`bereshit/word/seed/data/`

== Examples

* `config.toml`
* `data.json`
* `settings.yaml`
* Glossary term files (like this one)
