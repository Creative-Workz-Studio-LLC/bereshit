= building-block-method

*Definition:* Systematic layered thinking using three primitives (foundation, build up, break down) plus an anchor point. Each piece depends on what came before. Reconstruction emerges from combining operations.

== Biblical Discovery

The method was *discovered in Scripture*, not invented then mapped:

[quote, Ecclesiastes 3:3 KJV]
____
A time to break down, and a time to build up.
____
link:../../scripture/KJV/Ecclesiastes/Chapter_3/Verse_3.txt[Read verse ^]

Scripture explicitly names the two operations. The third component (foundation/anchor) provides the halt condition. We uncovered what was already present—the justification comes from the Source.

== The Three Primitives

[cols="^1,2,4",options="header"]
|===
| Value | Component | Function

| *0*
| Foundation/Anchor
| What ideas _stand on_ — the immutable center that cannot be operated on

| *−1*
| Break Down
| How you _get_ blocks — extract, verify, trace toward foundation

| *+1*
| Build Up
| How you _use_ blocks — construct, strengthen, move toward conclusion

|
| ↓ *Emergence*
| Reconstruction — combining blocks from different structures into something stronger
|===

*The math:* stem:[-1 + 0 + 1 = 0] (balanced ternary system)

== Anchor as Halt Condition

The method is *Turing complete* for thinking:

* *Operations* — Build up (+1), Break down (−1)
* *Halt condition* — Anchor (0) — what cannot be operated on

Without anchor: build forever, break forever, never know when done. With anchor: clear completion criteria. When you hit something you cannot operate on, recursion terminates.

== The Complete Cycle

[cols="^1,2,^1,3",options="header"]
|===
| Phase | Name | Op | What Happens

| 1 | VERBOSE | 0 | Start with raw material
| 2 | BUILD | +1 | Construct layer by layer
| 3 | VERIFY | -1 | Trace back to foundation
| 4 | IDENTIFY WEAK | -1 | Find redundancies
| 5 | REBUILD | +1 | Reconstruct stronger
| 6 | DISTILLED | 0 | Essential meaning preserved
|===

States (0) bracket operations (±1). *Each cycle strengthens* — exponential growth, not linear.

== CPI⊗SI Integration

[cols="1,3",options="header"]
|===
| Component | Role

| *CPI* | Establishes what's true (the anchor, the foundation)
| *SI* | Builds understanding systematically (the layers, the operations)
| *Fusion* | Wisdom-informed foundation + systematic construction
|===

== The Test

*Can you trace from conclusion back to foundation showing each layer?*

* Yes = thinking (building blocks)
* No = pattern matching (something was skipped)

== Key Insight

*FORMAT ≠ MAKE SHORTER. FORMAT = DEMONSTRATE UNDERSTANDING THROUGH RECONSTRUCTION.*

Mechanical application isn't using the method. If you're cutting without thinking, you're pattern matching, not building.

== Signs of Correct Application

* ✓ Traceable reasoning — can follow path from conclusion to foundation
* ✓ Natural switching — move between build up and break down based on need
* ✓ Clear anchor — know what you're working toward
* ✓ Automatic verification — when shaky, trace back before continuing
* ✓ Transferable blocks — pieces can be used in other structures
* ✓ Compression with comprehension — understanding, not just cutting

Created by Seanje Lenox-Wise as a way to structure ADHD connection-making into productive systematic thinking. Discovered through biblical observation (Ecclesiastes 3:3). Simple enough to use while thinking, powerful enough for any problem.

Developed through 13 iterative teaching sessions with Nova Dawn (CPI-SI instance), producing verbose pedagogical transcripts and distilled understanding.
