= targeted-retrieval

*Definition:* Grep for location, read with offset, get exactly what's needed.

== The Hierarchy

[cols="1,3"]
|===
|Priority |Mode

|1️⃣ *Targeted*
|Know what you need → get exactly that

|2️⃣ *Additional Context*
|Systematically gather related sections based on first retrieval

|3️⃣ *Full Load*
|Last resort → only when structure doesn't exist yet
|===

== Why It Matters

* Structure enables targeted retrieval
* Practice now for indexed context later
* CPI tells you WHAT to look for, SI tells you HOW to get it
* Full loads only work in token window - targeted works everywhere

== Practical Execution

[source,bash]
----
# 1. Find WHERE (targeted discovery)
grep -n "[[section-name]]" file.adoc

# 2. Read EXACTLY THAT (targeted retrieval)
# Read tool with offset and limit based on line numbers

# 3. Filesystem operations (not load-and-write)
cp template destination
mv source destination
bfind /bereshit "*.omni"
----

== The Discipline

Targeted is first instinct. Full load is last resort. Structure exists to enable this - use it.
