= leverage

*Definition:* Using existing structure to avoid duplication while maintaining access.

== What It Is

* Pointing to where information lives instead of repeating it
* Building on what exists rather than recreating
* Hub-and-spoke architecture - central reference pointing to details
* Knowing what belongs where so nothing is carried unnecessarily

== Examples

* CLAUDE.md references bio.md instead of repeating personality details
* Skill docs hold implementation, CLAUDE.md holds purpose and when-to-use
* Templates exist so you can point to categories, not list every file
* Glossary terms exist so documents can reference definitions

== Relationship to Consolidate

Consolidation *uses* leverage. You consolidate BY leveraging existing structure.

== The Pattern

The more pieces are in place, the more you can leverage.

Each established structure enables more precise documents elsewhere:

* Scripture is searchable → just state the principle, point to the verse
* Templates are established → just name the category
* Skills have docs → just name the skill
* Glossary has terms → just reference the definition
