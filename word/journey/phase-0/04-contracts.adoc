[[process]]
== Process: How We Built

=== 1. Identify Behavioral Domains

What components need behavioral contracts?

[cols="2,4",options="header"]
|===
| Component | Behaviors to Specify

| *Health Provider*
| Read, write, normalize, convert, display

| *Filesystem*
| Create, read, update, delete, atomic operations

| *Identity*
| Create, verify, relate

| *Network*
| Connect, timeout, retry, secure
|===

=== 2. Create Contract Files

[source,bash]
----
mkdir -p word/core/contracts

cp word/seed/data/toml/toml-data.toml word/core/contracts/health-provider.toml
cp word/seed/data/toml/toml-data.toml word/core/contracts/filesystem.toml
cp word/seed/data/toml/toml-data.toml word/core/contracts/identity.toml
cp word/seed/data/toml/toml-data.toml word/core/contracts/network.toml
----

=== 3. Define MUST/SHOULD/MAY Semantics

Following RFC 2119 semantics:

[cols="1,4,3",options="header"]
|===
| Level | Meaning | Example

| *MUST*
| Absolute requirement — violation is contract breach
| Health provider MUST normalize before storing

| *SHOULD*
| Strong recommendation — deviation requires justification
| Filesystem SHOULD use atomic writes

| *MAY*
| Optional behavior — implementer's choice
| Network MAY retry on transient failures
|===

=== 4. Ground in Scripture

Contracts are vows — "he shall not break his word." When a component promises behavior, it MUST keep that promise. A health provider that says "I WILL normalize before storing" must do so.

[source,toml]
----
[contract.health_provider]
name = "HealthProvider"
description = "Behavioral contract for health data operations"
biblical_anchor = "Numbers 30:2"

[[contract.health_provider.behaviors]]
name = "normalize_on_store"
level = "MUST"
description = "Normalize health values before storing"
rationale = "Ensures consistent data across the system"

[[contract.health_provider.behaviors]]
name = "validate_on_read"
level = "MUST"
description = "Validate health values when reading"
rationale = "Prevents invalid data from propagating"

[[contract.health_provider.behaviors]]
name = "cache_recent"
level = "SHOULD"
description = "Cache recently accessed health data"
rationale = "Performance optimization"

[[contract.health_provider.behaviors]]
name = "compress_history"
level = "MAY"
description = "Compress historical health data"
rationale = "Storage optimization, implementer's choice"
----

== Biblical Foundation

[NOTE]
====
*A contract is a vow.* When a component promises behavior, breaking that promise is breaking its word. In Kingdom Technology, we keep our promises.
====

_"If a man vow a vow unto the LORD, or swear an oath to bind his soul with a bond; he shall not break his word, he shall do according to all that proceedeth out of his mouth."_ — *Numbers 30:2 KJV*

_"When a man vows a vow to Yahweh, or swears an oath to bind his soul with a bond, he shall not break his word. He shall do according to all that proceeds out of his mouth."_ — *Numbers 30:2 WEB*

*Applied:* A contract is a vow. When a health provider says "I WILL normalize before storing," it must do so. Breaking a contract is breaking a vow — unacceptable in Kingdom Technology. The MUST/SHOULD/MAY levels reflect the weight of the promise.
