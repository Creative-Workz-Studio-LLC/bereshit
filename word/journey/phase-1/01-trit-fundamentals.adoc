== Purpose

This document describes *how we built trit.h* — the MATTER layer of libtrit. The trit type is the atomic building block: what things ARE, not how they relate (SPACE) or when they occur (TIME).

[[process]]
== Process: How We Built

=== 1. Map Trit Values to Dimensions

From Genesis 1:1, we mapped the three cognitive dimensions:

[cols="^1,2,3,3"]
|===
| Value | Dimension | Genesis 1:1 | Cognitive Meaning

| *-1*
| MATTER
| "the earth"
| What things ARE (break down)

| *0*
| TIME
| "in the beginning"
| When things happen (anchor)

| *+1*
| SPACE
| "the heavens"
| Where things relate (build up)
|===

=== 2. Design Type Hierarchy

We needed types at different scales:

[source]
----
trit_t     → single digit (-1, 0, +1)          → int8_t
trit5_t    → 5 trits packed (243 states)       → uint8_t
trit9_t    → 9 trits packed (19,683 states)    → uint16_t
trit27_t   → 27 trits packed (7.6T states)     → uint64_t
----

=== 3. Define Constants and Macros

Three categories of constants:

*Computational values:*
[source,c]
----
#define TRIT_NEG   (-1)
#define TRIT_ZERO  (0)
#define TRIT_POS   (+1)
----

*Dimensional aliases:*
[source,c]
----
#define TRIT_MATTER  TRIT_NEG   // "the earth"
#define TRIT_TIME    TRIT_ZERO  // "in the beginning"
#define TRIT_SPACE   TRIT_POS   // "the heavens"
----

*Conversion macros:*
[source,c]
----
#define TRIT_TO_UNSIGNED(t)   ((t) + 1)   // -1→0, 0→1, +1→2
#define UNSIGNED_TO_TRIT(u)   ((u) - 1)   // 0→-1, 1→0, 2→+1
----

=== 4. Precompute Power Arrays

For Horner's method pack/unpack, we precomputed 3^n values:

[source,c]
----
static const uint8_t TRIT5_POWERS[5] = {
    1, 3, 9, 27, 81
};

static const uint16_t TRIT9_POWERS[9] = {
    1, 3, 9, 27, 81, 243, 729, 2187, 6561
};

static const uint64_t TRIT27_POWERS[27] = {
    1ULL, 3ULL, 9ULL, 27ULL, /* ... */ 2541865828329ULL
};
----

=== 5. Declare Function Prototypes

Single trit operations (src/trit.c):
[source,c]
----
trit_t trit_create(int value);
bool trit_valid(trit_t t);
int trit_value(trit_t t);
trit_t trit_negate(trit_t t);
trit_t trit_add(trit_t a, trit_t b);
trit_t trit_multiply(trit_t a, trit_t b);
----

Pack/unpack operations (src/pack.c):
[source,c]
----
trit5_t trit5_pack(const trit_t trits[5]);
void trit5_unpack(trit5_t packed, trit_t trits[5]);
bool trit5_is_spare(trit5_t value);  // Bible Rail detection
----
