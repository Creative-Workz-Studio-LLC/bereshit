=== Phase 0 Outputs

Phase 0 produced the definitional layer:

[cols="2,4"]
|===
| Output | Content

| *word/core/primitives.toml*
| Primitive type definitions (Trit, HealthScore, etc.)

| *word/core/types.toml*
| Composite type definitions

| *word/core/schemas.toml*
| Validation schemas

| *word/core/contracts.toml*
| Behavioral contracts
|===

=== Research Foundation

Complete mathematical and algorithmic research:

[cols="2,4"]
|===
| Document | Content

| *ternary-mathematical-foundations.adoc*
| Radix economy proof — why base 3 is optimal

| *ternary-arithmetic-algorithms.adoc*
| Addition, subtraction, multiplication in balanced ternary

| *ternary-storage-algorithms.adoc*
| Packing trits into bytes, storage patterns

| *ternary-packing-algorithms.adoc*
| Horner's method proofs, roundtrip correctness
|===

----

│   ├── research/   # Algorithm research
│   ├── scripture/  # KJV and WEB

[[prerequisites]]
== Prerequisites for Reproduction

To reproduce Phase 1, you need:

=== 1. Phase 0 Outputs

The core/ folder must contain complete type definitions:

[source,bash]
----
word/core/
├── primitives.toml  # Primitive types (Trit, etc.)
├── types.toml       # Composite types
├── schemas.toml     # Validation rules
└── contracts.toml   # Behavioral contracts
----

=== 2. Research Documents

The research folder with algorithm specifications:

[source,bash]
----
word/research/ternary/
├── ternary-mathematical-foundations.adoc
├── ternary-arithmetic-algorithms.adoc
├── ternary-storage-algorithms.adoc
└── ternary-packing-algorithms.adoc
----

=== 3. Code Templates

The 4-block code template for C headers:

[source,bash]
----
word/seed/code/c/header.h
----

=== 4. Empty pkg/trit Folder

The target folder for the library:

[source,bash]
----
mkdir -p word/work/pkg/trit/{include,src,test}
----
