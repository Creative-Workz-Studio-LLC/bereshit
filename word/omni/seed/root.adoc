[[overview]]
== Overview

OmniCode Seeds is a child of word/omni/, containing σπέρμα (sperma/seeds) — template files specifically for creating new OmniCode (.omni) documents. These templates ensure consistency and proper structure in all OmniCode files.

This document follows the CWSfootnote:[CWS = CreativeWorkzStudio LLC] 5-block documentation structure.

[IMPORTANT]
====
*#OmniCode-specific templates#* — These seeds produce only OmniCode files (.omni). For non-OmniCode templates (Go, AsciiDoc, etc.), see `word/seed/`. Tight strictness — OmniCode templates must be followed exactly.
====

[NOTE]
====
*Scope:* This document covers the word/omni/seed/ folder containing OmniCode-only templates. For the parent language definition, see `word/omni/`.
====

[[structure]]
== Structure & Flow

OmniCode seeds derive from the language specification and produce new OmniCode definitions.

[[flow]]
=== The Template Chain

[mermaid]
----
flowchart TD
    S[word/omni/syntax] --> U[seed/universal]
    U --> C[seed/code]
    U --> D[seed/documentation]
    U --> F[seed/folder]
    style S fill:#gold
----

. *syntax* — Master OmniCode specification
. *universal* — Base template all others derive from
. *specialized* — Code, documentation, folder templates

[TIP]
====
*Template chain integrity.* Changes to universal cascade to all specialized templates. Specialized templates add domain-specific structure.
====

[[health-scoring]]
=== Health Scoring

OmniCode Seeds use the *Base100* health scoring system.

[cols="^1,2,3",options="header"]
|===
| State | Threshold | Meaning

| *Granted (1)*
| Above +50
| Templates aligned with syntax spec

| *Deferred (0)*
| Within ±50
| Templates need alignment review

| *Denied (-1)*
| Below -50
| Templates diverged from syntax
|===
